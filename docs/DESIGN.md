# PulseHub / Slint 产品 UI 规范

> 状态：阶段 4 实现基线  
> 适用范围：`pulsehub-config.exe` 及其 Slint 组件  
> 相关文档：[IMPLEMENTATION.md](./IMPLEMENTATION.md)

## 1. 目标与边界

PulseHub GUI 是鼠标配置编辑器，不是设备控制代理。它负责展示代理快照、编辑两套配置、校验草稿、提交配置并解释保存与应用结果；设备枚举、设备句柄、环境切换和硬件写入始终由 `pulsehub-agent.exe` 负责。

设计目标：

- 用户无需理解 HID、profile format 或配置修订号，也能安全完成 Office 与 CS2 两套配置。
- 任何时刻都能区分“当前正在编辑什么”“配置是否已保存”“设备是否已应用”。
- 设备离线或能力尚不可用时，允许完成可离线校验的编辑，但不制造“已应用”的错觉。
- 校验失败、修订冲突、部分应用和系统集成失败均有明确、可恢复的操作路径。
- 关闭主窗口后隐藏到系统托盘，可从托盘恢复；无论窗口是否可见，GUI 均不持有设备资源。

非目标：

- GUI 不直接访问 HID 设备，不参与设备所有权。
- GUI 可作为轻量托盘宿主常驻；设备控制与自动切换仍属于代理。
- 首版不支持宏、主题切换、窗口内多文档或任意数量的配置环境。
- 不复制 Claude/Anthropic 的品牌标志、名称、营销页面布局或授权字体。

## 2. 设计原则

### 2.1 状态先于装饰

状态差异不能只依赖颜色。重要状态同时使用图标、短标题和说明文字；错误必须给出下一步操作。

### 2.2 能力驱动

DPI、物理控制和可映射动作全部来自代理返回的设备能力。能力未知时显示加载、离线或不支持状态，不用静态选项伪装成设备能力。

### 2.3 保存不等于应用

`commit_config` 成功只表示配置已经持久化。硬件应用和登录启动同步是后续独立结果。界面禁止使用一个笼统的“成功”状态同时代表三者。

### 2.4 就地解释、就地恢复

字段错误靠近字段显示；页面级问题显示在内容顶部；跨页面或阻断性问题使用对话框。能原地重试时，不把用户送往诊断页面。

### 2.5 温和、克制、桌面化

沿用暖米色画布、暖黑文字和珊瑚色主操作，但采用紧凑的 Windows 桌面信息密度。主要层级依靠间距、色块和细边框，少用阴影，不使用营销式大标题或大面积 CTA。

## 3. 信息架构

首版窗口采用左侧导航、右侧内容、底部操作区。

```text
┌──────────────────────────────────────────────────────────────┐
│ PulseHub                                      设备状态 ···   │
├──────────────┬───────────────────────────────────────────────┤
│ 设备概览     │ 页面标题                         页面状态      │
│ Office 配置  │                                               │
│ CS2 配置     │ 页面内容                                      │
│ 设置         │                                               │
│              │                                               │
├──────────────┴───────────────────────────────────────────────┤
│ 保存/应用摘要                         放弃更改   保存配置    │
└──────────────────────────────────────────────────────────────┘
```

- 顶栏：产品名、当前设备摘要、代理连接状态、诊断入口。
- 左侧导航：设备概览、Office 配置、CS2 配置、设置。
- 内容区：当前页面标题、说明、页面级状态和表单。
- 底部操作区：只在存在可提交草稿或保存/应用状态需要解释时显示。
- 页面切换不丢弃草稿；Office、CS2 和设置共享一个待提交草稿。

默认窗口建议为 `960 × 680 px`，最小窗口为 `800 × 600 px`。窗口允许缩放，但首版不设计移动端布局。低于最小尺寸时不继续压缩控件。

## 4. 页面规范

### 4.1 设备概览

页面回答三个问题：设备是否可用、当前采用哪套环境、最后一次应用是否成功。

内容顺序：

1. 设备卡：连接状态、型号、固件或硬件修订（若可用）。
2. 当前环境：Office、CS2 或手动覆盖，并注明切换来源。
3. 当前设备值：实际 DPI、目标 DPI、轮询率。
4. 应用状态：最后应用时间、结果、公开错误码和重试入口。
5. 冲突提示：例如 G HUB 占用、设备不支持或代理不可用。

设备状态：

| 状态 | 标题 | 表现 | 可用操作 |
|---|---|---|---|
| `ready` | 设备已连接 | 绿色状态点 + 型号 | 查看或编辑配置 |
| `connecting` | 正在连接设备 | 进度指示，不阻塞导航 | 等待 |
| `disconnected` | 未检测到设备 | 中性图标 + 离线说明 | 重新检测、继续编辑草稿 |
| `busy` | 设备正被其他程序占用 | 黄色警告 | 提示退出 G HUB、重试 |
| `unsupported` | 不支持此设备 | 红色错误 | 查看诊断信息 |
| `agent_offline` | PulseHub 代理未运行 | 页面级错误 | 启动代理、重新连接 |

### 4.2 Office 配置

包含配置名称、DPI、轮询率（能力支持时）和按键映射。

- DPI 使用单选卡、分段选择或下拉框，选项只能来自能力集合。
- 选项较少（不超过 6 个）时优先使用横向选择；更多时使用下拉框。
- 当前实际 DPI 与草稿 DPI 不同时，在字段下显示“保存后将应用到设备”，不能把实际值改写成草稿值。
- 按键映射按设备物理布局或稳定控制顺序展示；每行包含控制名称、位置说明、当前动作和动作选择器。
- 主点击保护属于阻断校验；禁止配置时立即在对应行显示原因。

### 4.3 CS2 配置

沿用 Office 页面结构，增加进程规则。

- 默认规则清楚显示触发进程，例如 `cs2.exe`。
- 进程规则首版采用受约束的可编辑列表，不提供任意脚本或正则表达式。
- 展示“自动切换”是否启用；手动模式下仍可编辑，但说明不会自动激活。
- 若设备能力不支持安全的运行时按键切换，相关字段禁用并解释原因，不静默退化为频繁写入闪存。

### 4.4 设置

包含：

- 登录 Windows 时启动代理。
- 自动/手动环境选择；手动模式下选择 Office 或 CS2。
- 诊断日志状态、打开日志位置或复制诊断摘要。
- 版本与 IPC 协议信息。

登录启动同步失败时，配置仍可显示为“已保存”，但该设置旁显示“系统设置同步失败”，提供重试；不得回滚或伪装成未保存。

## 5. 核心状态模型

### 5.1 三层状态

UI 必须分别维护并展示：

| 层次 | 来源 | 含义 |
|---|---|---|
| 草稿状态 | GUI 本地 | 用户当前看到和编辑的值 |
| 保存状态 | `commit_config` 响应/快照 | 配置文件是否已经持久化 |
| 应用状态 | 代理事件/快照 | 目标配置是否已经写入当前设备 |

推荐状态字段：

```text
draft_dirty: bool
draft_validation: unknown | validating | valid | invalid
save_status: idle | saving | saved | failed | conflict
apply_status: unknown | pending | applying | applied | partial | failed | unavailable
integration_status: unknown | syncing | synced | failed
base_revision: u64
snapshot_revision: u64
```

不得根据按钮点击推测成功。只有代理响应或后续快照/事件才能推进保存、应用和系统集成状态。

### 5.2 用户可见文案

| 组合 | 主状态 | 辅助说明 |
|---|---|---|
| 无草稿，已应用 | 已应用 | 配置已保存并应用到设备 |
| 有草稿 | 有未保存的更改 | 保存后才会发送给代理 |
| 正在校验 | 正在检查配置 | 暂时禁用重复提交 |
| 保存中 | 正在保存 | 不显示已应用 |
| 已保存，等待应用 | 已保存，正在等待应用 | 配置文件已经更新 |
| 已保存，已应用 | 已应用 | 配置已保存并应用到设备 |
| 已保存，应用失败 | 已保存，未应用 | 更改不会丢失，可重新应用 |
| 已保存，设备离线 | 已保存，设备未连接 | 设备连接后由代理应用 |
| 部分应用 | 已保存，部分应用 | 明确列出成功项与失败项 |
| 修订冲突 | 配置已在其他窗口更新 | 重新载入后再编辑 |
| 保存失败 | 未能保存配置 | 草稿保留在当前窗口 |

成功绿色只用于“已应用”或“系统同步完成”。“已保存、未应用”使用黄色或中性状态，不能使用绿色对勾。

### 5.3 草稿校验

校验分为本地即时校验和代理校验：

1. 编辑时执行轻量本地校验，及时发现空值、重复规则和显然非法的选择。
2. 点击保存或触发“保存并关闭”时调用 `validate_draft`。
3. 校验通过后才调用 `commit_config(base_revision)`。
4. 返回字段错误时定位到第一个错误，并保留全部草稿。

错误显示规则：

- 字段边框使用错误色，字段下方显示一句可执行的修复说明。
- 页面顶部显示错误摘要及数量；点击摘要项滚动并聚焦字段。
- 不只显示内部字段名或 Rust/IPC 错误文本。
- 公开错误码可显示在详细信息中，例如 `PH-DEV-BUSY`。

### 5.4 修订冲突

当 `base_revision != config_revision` 时停止提交，显示阻断对话框：

**标题：** 配置已在其他窗口更新  
**正文：** 当前草稿基于旧版本。为避免覆盖新配置，请重新载入后检查更改。  
**操作：** `重新载入`（主要）、`保留草稿副本`（次要，如已实现）、`取消`。

首版若没有可靠的字段级合并，不提供“仍然覆盖”。重新载入前需要明确提醒本地草稿将被替换；若支持保留副本，应保留在内存或受控临时位置，不改变主配置。

## 6. 保存、关闭与最小化

### 6.1 保存按钮

- 无草稿：禁用，标签为“已保存”。
- 草稿无效：按钮可保持可点击以触发完整校验，也可禁用但必须显式显示错误；全项目保持一致。
- 校验/保存中：禁用并显示进度，防止重复提交。
- 提交成功：立即显示“已保存”，应用状态单独继续更新。
- 提交失败：恢复可点击，保留草稿。

### 6.2 关闭窗口

- 无脏数据：隐藏主窗口，托盘宿主继续运行。
- 有脏数据：显示“保存并关闭 / 放弃更改 / 取消”。
- “保存并关闭”先校验再提交；提交成功后隐藏窗口，不等待设备应用成功。
- 校验失败、修订冲突或保存失败：取消关闭，恢复窗口并定位问题。
- “放弃更改”只丢弃 GUI 草稿，不回滚已经保存的配置。

### 6.3 托盘与最小化

- 最小化使用标准窗口最小化行为；关闭按钮按 6.2 隐藏主窗口。
- 点击托盘图标或“打开 PulseHub”恢复并激活同一个窗口，同时刷新代理快照。
- 托盘菜单提供“退出托盘”；该操作退出整个 PulseHub：先请求代理应用 DPI `1600` 和全部原生按键，再安全结束代理与 GUI。
- 有脏数据时，“退出托盘”必须恢复窗口并走关闭确认流程，不能静默丢弃草稿。
- 托盘本身不直接访问 HID；安全退出配置必须由代理执行并回读校验。

| `primary` | `#C96F52` | 主要操作、焦点 |
| `primary_pressed` | `#A9583E` | 按下状态 |
| `primary_soft` | `#F4E3DC` | 主色弱背景 |
| `success` | `#3F8F57` | 已应用、已连接 |

所有文字与背景组合应满足 WCAG 2.1 AA；正文对比度至少 `4.5:1`，大文字和非文本关键控件至少 `3:1`。最终色值需在 Slint 实际渲染和 Windows 11 显示缩放下复核。

### 7.2 字体

```text
font-family: "Segoe UI Variable", "Microsoft YaHei UI", "Segoe UI", sans-serif
```

| 层级 | 字号 | 字重 | 行高 |
|---|---:|---:|---:|
| 页面标题 | 26px | 600 | 34px |
| 区块标题 | 18px | 600 | 26px |
| 控件标题 | 14px | 600 | 20px |
| 正文/控件 | 14px | 400 | 20px |
| 辅助文字 | 12px | 400 | 18px |
| 数值强调 | 20px | 600 | 28px |

中文界面不使用衬线展示字体。错误码、进程名和配置键可使用 `Cascadia Mono, Consolas, monospace`，但不应用于普通正文。

### 7.3 间距与尺寸

- 基础单位：4px。
- 页面外边距：24px。
- 区块间距：24px；相关字段组间距：16px；标签与控件：8px。
- 卡片内边距：16px 或 20px。
- 标准控件高度：36px；主要按钮可为 40px。
- 最小点击区域：40 × 40px。
- 输入框与按钮圆角：8px；卡片与对话框圆角：12px；状态胶囊圆角：999px。
- 边框：1px；键盘焦点环：2px 主色，并保留至少 1px 间隔。

### 7.4 阴影与动效

- 普通卡片不使用阴影，通过背景和边框分层。
- 对话框可使用低透明度阴影，但不能依赖阴影表达边界。
- 状态变化使用 120–180ms 的淡入或颜色过渡。
- 保存、应用和连接中的进度动画不得阻塞事件循环。
- 尊重 Windows 减少动画设置；关闭动效后状态仍完整可理解。

## 8. 组件规范

### 8.1 按钮

变体：`primary`、`secondary`、`ghost`、`danger`、`icon`。

每种按钮必须实现默认、悬停、按下、键盘焦点、禁用和忙碌状态。主操作每个操作区最多一个。破坏性操作不能仅靠红色区别，必须有明确动词，例如“放弃更改”。

### 8.2 选择控件

- DPI 和动作选择器禁止自由文本输入。
- 下拉项同时显示动作名称；需要时附一行兼容性说明。
- 不支持的动作原则上不出现；若为了说明能力差异而保留，必须禁用并解释原因。
- 使用键盘打开后支持方向键、Home/End、Enter 和 Escape。

### 8.3 按键映射行

每行包含：控制图示或编号、用户可理解的名称、当前动作选择器、错误/限制说明。整行高度建议不低于 52px。不要只展示 `G4` 等内部编号，应显示“侧后键（G4）”。

主点击、右键等安全关键项显示锁或保护提示。发生重复映射是否允许由领域规则决定，UI 不自行发明限制。

### 8.4 状态横幅

用于页面级的离线、设备忙、应用失败和修订变化。结构为图标、标题、最多两行说明、最多两个操作。横幅不自动消失；只有无需用户处理的短暂结果才可使用自动消失通知。

### 8.5 通知

短暂通知用于“诊断摘要已复制”等低风险确认。保存、应用、冲突和错误不能只依赖通知，因为用户可能错过。

### 8.6 对话框

仅用于关闭确认、放弃草稿、修订冲突和其他阻断决策。默认焦点放在安全操作上；Escape 等同取消；危险操作不设为默认按钮。

### 8.7 空状态与骨架状态

- 首次连接等待：显示骨架或进度及“正在连接 PulseHub 代理”。
- 无设备：保留配置页面入口，不把整个应用替换为空白页。
- 无能力：说明设备尚未连接，因此只能执行离线校验。
- IPC 断开：冻结最后快照并标注“信息可能已过期”，禁止提交直到重连。

## 9. 键盘与无障碍

- 所有功能可由键盘完成，Tab 顺序与视觉顺序一致。
- `Ctrl+S`：校验并保存；保存中忽略重复触发。
- `Esc`：关闭当前下拉或对话框；无浮层时不直接丢弃草稿。
- `Alt+F4`：执行标准关闭流程。
- 焦点不得被颜色变化或页面刷新悄然移走。
- 校验失败后聚焦第一个错误字段，并让屏幕阅读器获得错误摘要。
- 图标按钮提供中文可访问名称和工具提示。
- 状态点、图标和颜色之外必须有文字标签。
- 支持 Windows 100%、125%、150%、175% 和 200% 缩放；布局使用逻辑像素，不根据物理像素硬编码窗口内容。

## 10. Slint 实现约束

### 10.1 组件分层

建议结构：

```text
ui/
├─ app-window.slint
├─ theme.slint
├─ models.slint
├─ components/
│  ├─ app-shell.slint
│  ├─ buttons.slint
│  ├─ form-controls.slint
│  ├─ status-banner.slint
│  ├─ device-card.slint
│  ├─ mapping-row.slint
│  └─ dialogs.slint
└─ pages/
   ├─ overview-page.slint
   ├─ profile-page.slint
   └─ settings-page.slint
```

- 色彩、字号、间距、圆角和控件高度集中在 `theme.slint`，页面不得散落十六进制颜色。
- Slint 模型只承载展示所需数据；领域模型和 IPC DTO 不直接成为页面状态容器。
- Office 与 CS2 复用同一个 `ProfilePage`，通过模型和能力参数化，避免两份交互逐渐分叉。
- 所有用户意图通过 callback 传给 Rust，例如 `save-requested`、`validate-requested`、`retry-apply-requested`。

### 10.2 线程边界

- Slint 事件循环只运行在主线程。
- IPC、连接重试和其他阻塞工作只在工作线程执行。
- 工作线程通过 Slint 事件循环投递接口更新 UI，不直接修改组件。
- GUI 不创建 HID 句柄，也不链接或调用设备传输实现。
- 异步响应携带请求标识或草稿代次；过期校验结果不得覆盖较新的草稿状态。

### 10.3 快照映射

收到 `snapshot_changed` 时：

- 无本地草稿：用新快照更新表单和 `base_revision`。
- 有本地草稿且修订未变：只更新设备、应用等非编辑状态。
- 有本地草稿且修订已变：标记潜在冲突，不静默覆盖字段。
- `activate_ui`：恢复并前置现有窗口，不创建第二套页面状态。

### 10.4 生命周期

- 所有退出路径都停止 IPC 工作线程并等待有界清理，然后结束进程。
- 代理无响应不能导致 GUI 永久卡在退出中；超过时限后关闭 GUI 连接并退出。
- 不使用隐藏窗口维持 GUI 生命周期。
- 调试构建可记录生命周期事件，Release 不记录用户映射内容或敏感路径。

## 11. 文案规范

- 使用用户语言：“侧后键”“保存配置”“重新应用”，避免直接暴露 DTO、枚举名和内部状态。
- 状态标题不超过 12 个汉字，说明尽量不超过两行。
- 错误文案结构为“发生了什么 + 配置是否安全 + 下一步怎么做”。
- 不使用“OK”“失败了”“未知错误”等模糊文案。

示例：

- 推荐：“配置已保存，但鼠标当前未连接。设备重新连接后，PulseHub 会再次应用。”
- 推荐：“设备正被 G HUB 占用。请完全退出 G HUB，然后选择‘重新应用’。”
- 不推荐：“Apply failed: Busy(5)”。

## 12. 验收清单

### 12.1 功能与状态

- 能从代理能力动态生成 DPI、控制和动作选项。
- 无设备时可以编辑并保存通过离线校验的草稿，但不会显示“已应用”。
- `validate_draft` 失败时不调用 `commit_config`。
- `commit_config` 成功后正确区分等待应用、已应用、部分应用和应用失败。
- 登录启动同步失败独立显示，不回滚保存状态。
- 修订冲突不会静默覆盖新配置。
- IPC 断开与重连不会丢失或静默替换本地草稿。
- Office 与 CS2 两套配置均可仅用键盘完成编辑和保存。

### 12.2 生命周期

- 无草稿关闭后主窗口隐藏，托盘宿主继续运行。
- 有草稿选择保存并关闭，提交成功后主窗口隐藏。
- 校验、冲突或保存失败时窗口恢复且 GUI 不退出。
- 放弃更改后主窗口隐藏。
- 点击托盘可恢复同一个 GUI 实例，不产生多个无感知编辑器。
- 选择“退出托盘”后，代理先应用 DPI `1600` 和全部原生按键；成功后代理与 GUI 均退出。
- 设备在线但恢复失败时必须显示原因并允许重试或确认仍然退出；设备离线时有界等待后记录日志并退出。

### 12.3 视觉与无障碍

- 100%–200% Windows 缩放下无文本截断、控件重叠和不可达操作。
- 所有交互控件具备键盘焦点状态。
- 错误、警告、已保存和已应用不只依赖颜色区分。
- 中文、英文进程名和错误码混排不溢出。
- 主体文字与背景满足 WCAG AA 对比度目标。

## 13. 实施优先级

1. 先实现三层状态模型、窗口生命周期和无样式的完整流程。
2. 实现能力驱动的 Profile 页面及校验定位。
3. 实现修订冲突、IPC 断线和部分应用状态。
4. 建立主题令牌和共享组件，再完成四个页面视觉。
5. 完成 DPI 缩放、键盘、屏幕阅读器和 Windows 11 实机验收。

视觉完成不能替代状态验收。阶段 4 的最终完成标准仍是：用户可以安全完成两套配置，GUI 不参与设备所有权，关闭窗口后可由托盘恢复，退出托盘时安全还原鼠标并结束全部 PulseHub 进程。
