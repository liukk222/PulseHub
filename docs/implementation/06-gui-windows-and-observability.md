# PulseHub：GUI、Windows 集成与可观测性

> 涵盖 Slint GUI、系统托盘、Windows 集成、日志、安全与游戏兼容。

## 11. 配置 GUI

阶段 4 第一版 Slint 窗口已建立：默认启动 `pulsehub-config` 会打开 `960 × 680` 的桌面窗口，包含
设备概览、复用的 Office/CS2 配置页和设置页。主题令牌、状态横幅和设备卡按 `DESIGN.md` 分层，
Slint 事件循环留在主线程，IPC 快照读取、草稿验证与提交在工作线程执行，并通过事件循环投递结果。
草稿、保存和设备应用状态分别显示；提交成功不会被描述为设备已应用。

Slint 生成代码独立放在 `pulsehub-ui` crate。该 crate 是框架生成代码所需 unsafe 的唯一边界；
`pulsehub-config`、IPC、配置存储和设备实现继续执行 `forbid(unsafe_code)`。代理快照现已携带 DPI
最小值、最大值、步进和最多 6 个能力内快捷值；GUI 使用 Slint 模型动态生成选项。设备离线或探测
失败时不提供伪造选项，配置页保持可查看但无法选择未知能力。

GUI 在载入草稿时固定 `base_revision`，保存时直接提交该修订，不在提交前偷换为代理最新修订。
因此其他窗口先完成保存后，旧草稿会收到 `PH-IPC-CONFLICT`，保留草稿并显示“配置已在其他窗口
更新”。存在本地草稿时，“刷新”不会静默覆盖字段，必须先保存或显式放弃更改。

Office 与 CS2 配置页现从 `ConfigDocument.button_mappings` 构造独立 Slint 模型。左键和右键显示为
受保护且不可编辑；中键、侧键与 DPI 键可在已实机验证的 `Backspace / Ctrl+V / Ctrl+C /
Ctrl+A / 禁用` 白名单中切换。映射和 DPI 作为同一草稿先经代理 `validate_draft`，再按固定
`base_revision` 提交；GUI 仍不打开 HID 句柄，也不把保存成功描述为闪存已写入。

窗口关闭已接入 Slint `Window::on_close_requested`：无草稿时允许窗口隐藏并结束事件循环；有草稿时
保持窗口显示并打开阻断对话框，提供“保存并关闭 / 放弃更改 / 取消”。保存并关闭只有在代理完成
校验和提交后才退出；校验失败、修订冲突或 IPC 错误会取消关闭意图、保留草稿并恢复可操作状态。

设置页的环境选择已进入统一草稿，可在“自动 / 固定 Office / 固定 CS2”之间切换，并与 Profile
修改一次提交。代理下一次解析前台目标时立即读取新模式。`start_with_windows` 当前仅展示配置声明，
在 HKCU Run 同步与独立 `integration_status` 完成前保持不可编辑，避免把仅保存配置误报为系统集成成功。

`set_selection_mode` 已接入统一代理命令队列，通过 `ConfigRepository` 原子保存并递增修订，响应更新
目标环境与目标 DPI，但不把配置保存等同于 HID 应用。G102 实机联调完成 `auto -> cs2 -> auto`：
固定 CS2 时磁盘为 `mode = "cs2"`、快照为 `Cs2 / desired=100 / current=3200`；恢复自动后磁盘为
`mode = "auto"`、快照为 `Office / desired=3200 / current=3200`，验证模式变更未偷偷写设备。

`start_with_windows` 已接入 Windows 当前用户 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
的 `PulseHub` 值。配置提交成功后同步注册表，失败不回滚配置，并在快照中独立返回
`integration_status = synced | failed | unknown`。启动命令使用当前代理可执行文件的绝对路径及
`--run-agent --confirm-device-write`，不写 HKLM，也不要求管理员权限。

Windows 实机联调已完成禁用、启用和恢复：禁用后 Run 值不存在且快照为 `synced`；启用后 Run 值
包含代理绝对路径及上述参数，快照为 `synced`。测试结束后主配置与备份均恢复
`start_with_windows = true`，并删除测试创建的注册表值以恢复测试前系统状态。

设备概览页提供显式“重新应用”：仅在没有本地草稿且没有其他操作进行时可用，通过代理
`apply_now` 串行执行当前环境的完整设备应用。代理先校验运行态 DPI，再将当前环境 DPI 与六个按键
映射编码进同一个 G102 板载扇区事务，执行整扇区 CRC 与写后回读，最后启用板载模式。实机确认启用
板载模式后再次选择运行态 DPI 不会退出 `Onboard`，因此代理用该步骤校准活动 DPI。目标扇区已一致时
跳过闪存写入；IPC 为该显式事务保留 15 秒响应窗口。该操作不提交配置，但可能写入板载闪存。

2026-07-20 实机验证 Office 完整应用：独立 HID++ 会话回读为 `mode=Onboard`、当前 DPI `3200`、
板载 DPI 槽 `3200`；六个槽位依次为左键、右键、`Backspace`、`Ctrl+V`、`Ctrl+C`、`Ctrl+A`。
这修复了旧实现只调用 `set_first_g102_dpi`、从未把 GUI 映射发送到设备的问题。由于该型号只有一个
用户板载配置，Office/CS2 按键不同就无法只靠配置槽切换。自动模式现会同步完整按键配置，但通过 1 秒
稳定窗口、容量为一的最终目标合并、5 秒应用冷却和扇区内容幂等比较限制闪存写入；用户点击“重新应用”
仍可显式绕过环境等待并立即校验当前配置。

首次创建 schema v1 配置时，Office 与 CS2 均保留鼠标原始按键：左键、右键、中键、侧后、侧前和
DPI 切换。编辑器动作白名单增加按物理控制对应的“还原原本按键”选项，例如侧后键显示
“鼠标侧键（后）”、侧前键显示“鼠标侧键（前）”。默认值变更只影响首次生成配置，不覆盖已有用户
配置；已有配置可在映射行循环选择原本动作后保存。

Office 与 CS2 配置页使用纵向 `ScrollView`，内容高度超过窗口可视区时显示滚动条并响应滚轮；横向
滚动被禁用。小窗口不再裁掉靠下的 G4、G5、G6 映射行。

按键映射编辑器不再循环预设动作，也不再显示“点击更改”。每个未保护的映射行直接提供两个入口：

1. “输入按键组合”：进入独占键盘捕获，支持单个常规按键，或 `Ctrl / Shift / Alt / Win` 修饰键
   加一个主键；界面实时显示预览，`Enter` 将结构化 `usage + modifiers` 写入 GUI 草稿，`Esc` 取消。
2. “还原按键”：按物理槽位恢复中键、侧后、侧前或 DPI Cycle 的原始鼠标动作。

捕获动作仍需点击“保存配置”才进入配置仓库，再由“重新应用”显式写入设备。左、右主点击继续受
安全保护。G102 profile format `0x04` 的单键动作只能表示一个主键及多个修饰键，不允许把多个普通
主键伪装成宏；不支持的键会保留原草稿并显示错误。

DPI 快捷选项下方提供数字输入框。用户可按 Enter 或点击“确认”把数值写入当前 Profile 草稿；GUI
使用代理能力快照中的 `minimum / maximum / step` 校验，而不是只接受预设按钮。例如实测 G102 的
能力为 `50–8000 / step 50`，因此 `1250` 有效、`1255` 会被拒绝。自定义值同样遵循“先保存、再
重新应用”的事务边界。

当 G6 映射为“原本 DPI 切换”时，Profile 页面额外显示四个可编辑档位。schema v1 对旧配置采用
向后兼容默认值 `[800, 1600, 2400, 3200]`，保存时要求四项严格递增；每项还必须符合设备能力
范围与步进。显式“重新应用”会把四档写入板载 profile 的前四个 DPI slot、清空第五 slot，并把
当前 Profile DPI 对应的 slot 设为默认索引，因此 G6 在且仅在这四档间循环。若 G6 已改为键盘动作，
档位编辑区隐藏，设备层继续使用单一 DPI 写入路径，不要求当前 DPI 属于四档。

旧配置可能保存了 `1800` 等不在新默认四档中的当前 DPI。加载迁移会保留当前 DPI，用它替换距离
最近的默认档位并重新排序，例如 `1800 + [800,1600,2400,3200]` 迁移为
`[800,1800,2400,3200]`。这样不会把配置错误误报成设备忙，也不会要求用户退出未运行的 G HUB。
GUI 仅在 IPC 明确返回忙状态时提示检查其他鼠标管理程序；配置或协议错误按原始内容提示。

G102 实机还确认：`SET_ONBOARD_MODE` 会先激活 slot 0，即使 profile 的 `default_dpi_index` 指向其他
档位。完整应用因此在启用板载模式后再发送一次运行态目标 DPI，并用该次回读更新 IPC 快照；设备
保持 `mode=Onboard`。修复后 `1800 + [800,1800,2400,3200]` 的独立会话回读为当前 `1800`、
四档不变、板载模式有效。

GUI 使用 Slint，仅在主线程运行 Slint 事件循环。所有 IPC 均在工作线程完成，再通过 Slint 的事件循环投递接口更新 UI。

首版页面：

1. 设备概览：连接状态、型号、能力和冲突错误。
2. Office 配置：DPI 和按键映射。
3. CS2 配置：DPI、按键映射和进程规则。
4. 设置：登录启动、自动/手动切换、诊断日志。

交互规则：

- DPI 控件由能力集合生成，不显示任意自由输入值。
- 不受支持的动作不出现在按键下拉列表。
- 保存前先调用 `validate_draft`。
- 保存成功但硬件应用失败时，明确显示“已保存、未应用”。
- GUI 由用户直接启动但代理未运行时，可用绝对安装路径启动代理并在有限时间内重连。

关闭主窗口后保留 Slint 系统托盘宿主：

1. 页面无脏数据时隐藏主窗口，后台代理继续自动切换环境。
2. 页面有脏数据时显示“保存并关闭 / 放弃更改 / 取消”；保存失败时恢复窗口并定位错误。
3. 点击托盘图标或“打开 PulseHub”恢复现有窗口并刷新代理快照，不创建第二个编辑器。
4. 选择“退出托盘”时结束配置进程；若仍有脏数据，先恢复窗口并显示关闭确认。
5. 配置进程退出不影响独立运行的 `pulsehub-agent.exe`。

## 12. Windows 集成

| 能力 | API/机制 | 实现要点 |
|---|---|---|
| 托盘 | Slint `SystemTrayIcon` | 点击图标或菜单恢复窗口；“退出托盘”只结束配置进程 |
| Explorer 恢复 | `RegisterWindowMessageW("TaskbarCreated")` | 收到后重新添加图标 |
| 阻塞消息循环 | `GetMessageW / TranslateMessage / DispatchMessageW` | 正确区分 `0`、`-1` 和普通消息 |
| 前台变化 | `SetWinEventHook` | `WINEVENT_OUTOFCONTEXT \| WINEVENT_SKIPOWNPROCESS`；注册线程必须有消息循环 |
| 设备变化 | `RegisterDeviceNotificationW` | 窗口过程只排队，枚举在工作线程执行 |
| 电源恢复 | `WM_POWERBROADCAST / PBT_APMRESUMEAUTOMATIC` | 每个恢复代次只废弃一次旧句柄并重新枚举；不因后续 `PBT_APMRESUMESUSPEND` 重放 |
| 单实例 | 当前会话命名互斥体 | 名称包含 logon SID，不跨用户互斥 |
| 登录启动 | `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` | 每用户、无需提权；代理按配置同步，安装器卸载时清理 |

代理不是 Windows Service，因为托盘、前台窗口和每用户配置都属于交互式用户会话。

## 13. 错误处理与可观测性

### 13.1 错误分类

| 类别 | 例子 | 策略 |
|---|---|---|
| `Unsupported` | 型号、功能或动作不支持 | 不重试；提示用户 |
| `Validation` | DPI 越界、非法映射、版本冲突 | 不写设备和文件 |
| `Busy` | G HUB 占用或访问被拒绝 | 有界退避；提示退出 G HUB |
| `Disconnected` | 热拔插、睡眠后句柄失效 | 清空连接状态，等待设备事件 |
| `Protocol` | 超时、短包、错误 feature index | 重建会话后有限重试 |
| `PartialApply` | DPI 成功、按键失败 | 标记降级；不更新已应用指纹 |
| `ConfigIo` | 文件损坏、替换失败 | 保留旧文件；尝试备份 |
| `Ipc` | 版本不兼容、超长帧 | 断开该客户端，不影响代理 |

所有 Win32/HID 句柄使用 RAII 封装；Drop 负责关闭句柄，显式 shutdown 负责有序停止业务。

### 13.2 日志

- `agent.developer_logging` 控制开发者日志，schema v1 缺省值为 `false`；旧配置未包含字段时按关闭加载。
  设置页可随其他草稿保存该开关，统一代理提交后立即更新日志状态，无需重启。
- 开启后记录生命周期、设备连接、配置修订、切换结果和错误码。
- 不记录键盘输入内容、窗口标题或完整用户文件路径。
- 不记录鼠标移动报告。
- 协议原始报文仅在用户显式启用的诊断会话中记录。
- 代理日志保存在 `%APPDATA%\PulseHub\logs`，按 UTC Unix 日编号生成
  `pulsehub-agent-<day>.log`；记录 Unix 秒时间戳、级别和事件，不额外创建常驻日志线程。
- 每次代理启动都会扫描一次日志目录；运行期间每满 10 天再扫描一次。只对严格匹配上述命名规则的
  普通文件检查最后修改时间，超过 30 天时使用 `remove_file` 永久删除；不递归、不跟随符号链接，也不
  删除配置、备份或其他文件。文件筛选和物理删除范围已有单元测试覆盖。
- UI 展示稳定的公开错误码，例如 `PH-DEV-BUSY`，详细 Win32/HID 错误留在本地日志。

## 14. 安全与游戏兼容

- 以普通用户权限运行；不安装驱动、不注入进程、不请求 `SeDebugPrivilege`。
- 只用前台窗口事件识别环境，不读取游戏内存。
- 不安装 `WH_MOUSE_LL` / `WH_KEYBOARD_LL` 钩子。
- 不使用 `SendInput` 生成游戏操作。
- 只实现硬件支持的一对一映射，不实现宏。
- Named Pipe 只允许当前登录会话访问，且限制帧长和字段数量。
- 启动另一个可执行文件时使用安装目录中的绝对路径，并通过 `std::process::Command` 逐项传参，不拼接 shell 命令。
- 配置和日志保存在当前用户目录，不接受 IPC 传入任意输出路径。
- 对逆向得到的协议值进行严格边界检查，任何未知响应都按错误处理。

这些约束降低兼容和反作弊风险，但不构成对第三方游戏或反作弊系统的保证。发布前仍需核对 Valve/CS2 当时有效的规则。
